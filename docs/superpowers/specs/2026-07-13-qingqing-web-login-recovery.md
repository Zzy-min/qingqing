# 轻青 Web 登录恢复设计

## 问题

Flutter Web 登录页在桌面浏览器中只显示邮箱输入框，主操作按钮未可靠绘制，用户无法继续验证码登录。静态站点同时注册了 PWA service worker，可能继续提供旧构建。

## 目标

- 登录页首屏始终显示唯一主操作“获取验证码”。
- 邮箱输入框支持回车提交。
- 获取验证码后显示六位验证码输入框和“登录并继续”。
- 忙碌、错误与禁用状态保持明确。
- 本机开发仍由后端返回开发验证码，不绕过服务端会话授权。
- Web 构建不注册离线缓存，避免本地开发反复加载旧 UI。

## 非目标

- 不新增注册、密码登录、第三方 OAuth 或正式邮件服务。
- 不改变 VIP、BYOK 或模型权限逻辑。

## 验收

- 1440x900 与 390x844 下主按钮可见、可聚焦、可点击。
- 点击或在邮箱框按回车都可进入验证码步骤。
- 最新静态构建不包含活跃的 Flutter service worker 缓存策略。
